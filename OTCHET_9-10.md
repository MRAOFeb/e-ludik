## ОРГАНИЗАЦИЯ РАЗРАБОТКИ

### Выбор методологии управления процессом разработки

Для управления проектом E-LUDIK была выбрана методология Scrum — гибкий подход (Agile), ориентированный на итеративную разработку и быстрые поставки рабочего результата.

**Основные принципы Scrum:**

- **Спринты:** разработка разбивалась на короткие итерации с фокусом на конкретных функциях и исправлении ошибок.
- **Роли:**
  - Scrum Master — контролировал рабочий процесс и устранял препятствия.
  - Product Owner — формировал приоритеты задач и требования к функционалу.
  - Команда разработчиков — отвечала за реализацию пользовательских историй.
- **Артефакты:**
  - Бэклог продукта — полный список требований и улучшений для E-LUDIK.
  - Бэклог спринта — выбранный набор задач для текущей итерации.
  - Инкремент — рабочая версия веб-приложения, доступная для запуска и тестирования.

Scrum был выбран из-за потребности в быстрой адаптации к изменениям в требованиях, необходимости контроля сроков и эффективного распределения задач между участниками команды.

Для управления задачами и отслеживания прогресса использовался GitHub Issues и простая kanban-доска внутри репозитория.

### Стек технологий

Проект E-LUDIK представляет собой веб-приложение для социальной сети ставок и был реализован с использованием следующих технологий.

#### Веб-сервер и логика приложения

**Язык программирования:**
- Python — используется для разработки серверной логики и обработки пользовательских данных.

**Фреймворк:**
- Bottle — легковесный веб-фреймворк для маршрутизации, обработки форм и шаблонизации.

**База данных:**
- SQLite — встроенная реляционная база данных для локального хранения пользователей, ставок и связей между друзьями.

#### Клиентская часть

**Технологии:**
- HTML, CSS, JavaScript — интерфейс приложения реализован при помощи стандартных веб-технологий.
- Одностраничные страницы с шаблонами Bottle в каталоге `views/`.
- Статические ресурсы (`styles.css`, `app.js`, `default-avatar.svg`) лежат в папке `static/`.

#### Инструменты разработки

- Git — система контроля версий.
- GitHub — удалённый репозиторий и платформа для совместной работы.
- Python virtualenv — для локального управления зависимостями.
- `pip` — установка зависимостей из `requirements.txt`.
- MkDocs — генератор документации разработчика из Markdown.
- GitHub Wiki — платформа для размещения пользовательской документации прямо в репозитории.

#### Дополнительные библиотеки

- `bottle==0.13.4` — основной dependency для запуска веб-приложения.

## ВЫВОД

В ходе выполнения практических работ №9-10 была организована разработка веб-приложения E-LUDIK, ориентированного на социальную сеть ставок и анализ личных результатов.

Командой выбран гибкий подход Scrum, что позволило распределить задачи, быстро реагировать на изменения требований и последовательно развивать проект.

Серверная часть реализована на Python с использованием Bottle и SQLite, что обеспечило компактную структуру, простоту развертывания и понятную архитектуру.

Для документации разработчика использовали MkDocs, а для пользовательской документации — GitHub Wiki. Такое сочетание позволило быстро собрать сайт документации и хранить пользовательские инструкции прямо в репозитории.

Документация разработчика и пользователя охватывает ключевые аспекты проекта: структуру файлов, основные маршруты, взаимодействие с базой данных, запуск приложения и пользовательский интерфейс.
